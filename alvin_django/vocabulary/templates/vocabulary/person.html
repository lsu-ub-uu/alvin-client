Person

Authority records for persons are used in Alvin for:

  • Connecting the person as an agent via a role to an object or collection
  • Connecting the person as the subject of an object or collection

Authority work means controlled and structured handling of name forms. Users should be able to search for name forms in a uniform way. When persons change their name or act under a pseudonym, the user should be able to find the correct one from different name forms.

It means creating records as linked entities. This involves choosing a preferred name and, in some cases, distinctive additions to make the name unique. An important part of the work is to search for and check if an existing name form already exists as a linkable entity and, if so, link to it.

A person is defined as an individual or identity established by an individual (either alone or in collaboration with one or more other individuals). The term also includes people in sacred scriptures, fictional characters, and non-human entities, such as animal actors. It is also possible to register a record for an entire family. Associations of persons, such as legal entities and musical groups, are treated as Organisations.

Text information in notes etc. can be in any language.

To supplement the linked entities information about contributors as it appears on the item can be added in bibliographical records in free text as a note of the type Statement of responsibility.

Example: Kevin Wicker ; with a foreword by David Pritchard ; illustrated by Karel Feuerstein.

Sections in the registration form

At least one preferred name form of the person must be specified for the web pages in English, Swedish or Norwegian. This language specifies the language version of the web page rather than the language of the name form. If only one name form is specified, it will be displayed on all language versions of the web pages.

Some names have a language-specific form. This applies, for example, to saints, royalty, popes, bishops, etc., classical Greek, Roman and Byzantine persons, and people active before the year 1400.

The preferred name is the name by which the person is best known. To determine it, use the following sources (in order of priority):

  1. the preferred sources of information in items associated with the person, such as title pages
  2. other formal statements appearing in items associated with the person
  3. other sources (including reference sources such as VIAF or Wikipedia).

For people who change their name, the most recent name is generally chosen as the preferred name.

A person can have more than one identity, in which case multiple records are created as linked entities.

It is possible, but not mandatory, to specify the authorised form in all three display languages ​​in cases where they are different, eg. Chekhov, Anton/Tsjekhov, Anton/Tjechov, Anton.

The preferred name form must be in Latin script.

Do not enter more than one name for the same language.

Do not enter alternative name forms in this section.

Authority

Language code, display language:

At least one preferred name form of the person must be specified for the web pages in English, Swedish or Norwegian. This language specifies the language version of the web page rather than the language of the name form. If only one name form is specified, it will be displayed on all language versions of the web pages.

Family name:

Enter the authoritative family name form of the person corresponding to the selected language.

Example: Chekhov (in English). (Do not enter Чехов).

Leave empty for persons without a family name, eg. royalty or popes.

Given name:

Enter the authoritative given name form of the person corresponding to the selected language.

Example: Anton (in English). (Do not enter Антон).

Example: Charles (for Charles II, King of England, no family name).

Example: Socrates (classical Greek, no family name).

Numeration:

Enter ordinal numbers for royalty, popes, and bishops using Roman numerals.

Example: II (for Charles II, King of England).

Example: II Adolf (for Gustav II Adolf, King of Sweden).

Addition:

For royal titles, titles of nobility, titles denoting ecclesiastical rank, and religious titles. All other titles are only added if they are needed to distinguish one person from another with the same name.

Enter the title of royalty or nobility and the name of the state or people over whom the person rules or has ruled in the corresponding selected language.

Example: King of England (for Charles II, King of England).

Add titles for popes, bishops, other religious professions and saints in the corresponding selected language.

Example: Pope.

Add words etc. indicating relationship.

Example: Jr.

Do not enter a profession or occupation unless it is needed to distinguish people with the same name. Add such information in Field of Endeavour below instead.

Family:

Select Yes only if the record describes an entire family. Otherwise leave it unselected.

Enter the name of the family in the Family name field. Given name must be left empty.

Date of birth:

Date of death:

Enter date of birth and/or date of death.

All types of time data are specified according to the ISO 8601 standard according to the pattern yyyy (year), yyyy-mm (year and month) and yyyy-mm-dd (year, month and day).

Example: 1707 (year) 05 (month) 13 (day).

Example: 0470 (year).

Era (BC):

Enter BC for years before Christ. Otherwise leave unselected.

Example: 0470 (year) + BC (the year 470 BC).

Year/date displayed

Enter uncertain and approximate time information as free text. Where applicable, the year of birth/death can also be entered above for better searchability.

Example: 1717? (? is used for probable year).

Example: 1717 or 1718 (or is used for alternative and immediately following years).

Example: ca 1717 (ca is used for approximate year. Here the time is more fluid and can cover a time span of up to ten years).

Example: ca 1717-1767 (for approximate lifetime).

In all these cases, 1717 can also be entered as a numerical year of birth above, which means that 1717 will also be searchable as a year.

Example: Between 510 and 323 BC (as a time period).

Do not use this element if the exact date is known.

Place of birth:

Place of death:

Search for the authority record for the place of birth and/or death and link to the person. Create a new place record if needed.

Uncertain information:

Enter Uncertain information if the information is probable but not verfied. Otherwise leave unselected.

Nationality:

From the list, select nationality (country) or country of operation associated with the person. The 20 most common options are listed first. After these the others follow alphabetically.

Countries and kingdoms that no longer exist as independent states cannot be specified here.

More than one country may be selected.

Gender:

Select the person's gender from the list. If more than one, the most recent one is chosen.

A name that is not considered the authoritative form, e.g. alternative spellings, previous variants or other languages. Specifying alternative names makes the resource searchable using these. More than one alternative may be added.

Variant

Language code:

Select the language corresponding to the name form. Any language can be specified. The 20 most common options are listed first. After these the others follow alphabetically.

Variant:

Select the type of variant form. Alternative name is selected if none of the more specified ones are applicable.

Family name:

Enter the variant family name form of the person corresponding to the selected language.

Example: Чехов (in Russian, Cyrillic script).

Given name:

Enter the variant given name form of the person corresponding to the selected language.

Example: Антон (in Russian, Cyrillic script).

Numeration:

Enter ordinal numbers for royalty, popes, and bishops using Roman numerals.

Example: II (for Oscar II, roi de Suède et de Norvège, in French).

Addition:

Enter the title of royalty or nobility and the name of the state or people over whom the person rules or has ruled in the corresponding selected language.

Example: roi de Suède et de Norvège (for Oscar II, roi de Suède et de Norvège, in French)

Add words etc. indicating relationship.

Example: Jr.

Text direction:

Select Right-to-left only if the name should be displayed from right to left, e.g, names in Arabic or Hebrew. Otherwise leave it unselected.

Field of activity:

Words or phrases that describe a person's profession or occupation. They do not have to be related to professional practice.

Example: poet

Example: amateur archaeologist

Example: artist

Note:

Type:

Select the type of note from the list. Enter only one note for each type (all information in one element).

  • General - General information for which a specialised note field has not been defined.
  • Biographical/historical - Biographical information about an individual or historical information about an institution.
  • Source - Citation for a consulted source in which information is found related in some manner to the entity represented by the authority record or related entities, for example a bibliographic reference or a URL.

Example: Born in Skogs socken, Hälsingland, the son of chaplain Pehr Ahl and Elisabeth Grönlund. Doctor of medicine 1789, town physician in Sala from 1796 (biographical).

Identifier:

Type:

Select the type of identifier from the list.

Example: 34594730 (VIAF for Carl von Linné).

Enter the number/code only, not the full URL. A link to the source will automatically be created on the web page.

Related person

Search for the authority record for the related person and link to the person described in the record. Create a new person record if needed.

Type of relation cannot be specified here but can be added as a general note, for example Married to Ingeborg Abbott (b. Stephens) (as a note).

Related organisation

Search for the authority record for the related organisation and link to the person described in the record. Create a new organisation record if needed.

Type of relation cannot be specified here but can be added as a general note, for example Doctor of history and political science at Uppsala. Librarian at the university library. (Uppsala university).

External electronic location (URL)

URL:

An external link that is relevant to the record described.

The URL must be correctly encoded according to the RFC 3986 standard and start with http:// or https://.

Example: https://en.wikipedia.org/wiki/Carl_Linnaeus

It is recommended that all URLs used with the element are persistent.

Display label:

A descriptive label displayed on the web page.

Example: Wikipedia
