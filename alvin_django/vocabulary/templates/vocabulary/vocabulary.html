{% translate 'Basvokabulär' %}

{% translate 'För att kunna hantera olika typer av resurser i ett och samma system bygger den på en kombination av koncept från olika internationella standarder för bibliotek och arkiv:' %}

BIBFRAME (Bibliographic Framework Initiative)
{% translate 'Grundläggande vokabulär för allmänna egenskaper som titel, språk och ursprungsinformation.' %}
MODS (Metadata Object Description Schema)
MARC (MAchine-Readable Cataloging) 21 Format for Bibliographic Data
{% translate 'För mer specifika bibliografiska egenskaper som saknas i BIBFRAME.' %}
EAD (Encoded Archival Description)
{% translate 'För arkivbeskrivningar.' %}
TEI (Text Encoding Initiative, Manuscript Description)
{% translate 'För mer specifika egenskaper för handskrifter.' %}
NUDS (Numismatic Description Schema)
{% translate 'För specifika egenskaper inom numismatik.' %}
MADS (Metadata Authority Description Schema)
{% translate 'För auktoritetposter för person, organisation och plats.' %}
METS (Metadata Encoding and Transmission Standard)
{% translate 'För beskrivningar av en posts relaterade digitala filer.' %}
WGS 84 (World Geodetic System 1984)
{% translate 'För att ange geografiska koordinater (latitud och longitud).' %}
ISO 8601
{% translate 'För att ange datum och tid.' %}

{% translate 'Kärnklasser' %}

{% translate 'Modellen är uppbyggd kring fem grundläggande kärnklasser (core classes):' %}

  • record - {% translate 'en bibliografisk post som beskriver ett enskilt objekt eller en samling av objekt.' %}
  • person - {% translate 'en auktoritetspost för en person.' %}
  • organisation - {% translate 'en auktoritetspost för en organisation.' %}
  • place - {% translate 'en auktoritetspost för en plats.' %}
  • work - {% translate 'en auktoritetspost för ett verk.' %}

{% translate 'Samt förutom dessa även:' %}

  • location - {% translate 'en särskild organisationspost som beskriver den arkivinstitution som innehar föremålet eller från vilken det är tillgängligt.' %}

{% translate 'Subklasser och egenskaper' %}

{% translate 'Kärnklasserna innehåller i sin tur olika uppsättningar av:' %}

  • subclass - {% translate 'en grupp som samlar en eller flera klasser eller egenskaper, till exempel title som har egenskaperna mainTitle och subtitle.' %}
  • property (text) - {% translate 'ett värde som har skrivits in, till exempel en titel.' %}
  • property (code) - {% translate 'ett värde valt från en kontrollerad lista i Alvin, till exempel språk.' %}
  • property (link) - {% translate 'en länk till en post från en kärnklass, till exempel från en bibliografisk post till en auktoritetspost för en person.' %}

{% translate 'Länkad data' %}

{% translate 'Modellen är baserad på länkad data och beskrivningen av relationer mellan olika poster. Länkningen kan vara hierarkisk (värd/del), parallell (olika versioner eller format) eller till en annan relaterad post.' %}

{% translate 'Bibliografiska poster och auktoritetsposter är länkade genom en typ av relation (roll) för så kallade agenter (personer eller organisationer). En bok kan länkas till en person genom rollen "författare". Ett arkiv kan länkas till en person genom rollen "arkivbildare".' %}

{% translate 'En bibliografisk post i Alvin beskriver antingen en enskild resurs (bok, bild, karta etc.) eller en samling resurser (t.ex. ett arkiv).' %}

{% translate 'Objekt' %}

{% translate 'Det vanligaste fallet är att ett fysiskt objekt beskrivs. Det kan till exempel vara en målning, en karta, ett manuskript eller en tryckt bok. Posten i Alvin baseras på det fysiska exemplar som beskrivs. För tryckt eller annat material som reproduceras i utgåvor beskrivs fortfarande det fysiska exemplaret i Alvin till skillnad från till exempel bibliotekskataloger. Om man har olika exemplar av en bok från samma utgåva finns det en post per exemplar. Proveniens, förvärv, signaturer etc. gäller för just detta exemplar i Alvin.' %}

{% translate 'Samling' %}

{% translate 'I Alvin kan, förutom konkreta objekt, även abstrakta resurser beskrivas. Exempel på detta är främst samlingar eller andra översiktliga beskrivningar. Dessa är ofta, men inte alltid, endast abstrakta konstruktioner som i sin tur innehåller konkreta objekt som ibland beskrivs separat som ovan och ibland inte. Emellertid kan en samlingspost också betraktas som en heltäckande beskrivning av det konkreta innehållet. Alvin tillåter att digitalt material även länkas till abstrakta poster. Man kan välja att ladda upp alla bilder i en bildsamling till huvud-/samlingsposten utan att beskriva de olika delarna individuellt.' %}

{% translate 'Digitala reproduktioner' %}

{% translate 'Normalfallet i Alvin är också att göra en digital reproduktion av det fysiska objektet, t.ex. digitalisera en handskrift eller en bok. I dessa fall beskriver metadataposten fortfarande det fysiska objektet (inte den digitala reproduktionen). De digitala filerna är kopior av just detta exemplar som beskrivs i posten.' %}
