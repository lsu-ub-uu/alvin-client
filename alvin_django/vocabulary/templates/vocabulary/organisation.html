Organisation

Authority records for organisations are used in Alvin for:

  • Connecting the organisation as an agent via a role to an object or collection
  • Connecting the organisation as the subject of an object or collection

Authority work means controlled and structured handling of name forms. Users should be able to search for name forms in a uniform way.

It means creating records as linked entities. This involves choosing a preferred name and, in some cases, distinctive additions to make the name unique. An important part of the work is to search for and check if an existing name form already exists as a linkable entity and, if so, link to it.

An organisation is defined as a corporation or group of persons and/or organisations that acts, or may act, as a unit.

Examples of organisations are associations, companies, nonprofit organisations, governments, state agencies, municipalities, projects, music groups, religious communities and conferences.

Temporary events such as sports competitions, exhibitions, expeditions, fairs and festivals, as well as vehicles, such as ships and spacecraft, are also considered organisations.

Text information in notes etc. can be in any language.

Sections in the registration form

At least one preferred name form of the organisation must be specified for the web pages in English, Swedish or Norwegian. This language specifies the language version of the web page rather than the language of the name form. If only one name form is specified, it will be displayed on all language versions of the web pages.

For foreign organisations that lack an accepted name form in any of the display languages, select the name form from the respective country together with the display language.

Some names have a language-specific form. This applies, for example, to international organisations, older organisations and religious orders and communities.

The preferred name is the name by which the organisation is best known. To determine it, use the following sources (in order of priority):

  1. the preferred sources of information in items associated with the organisation, such as title pages
  2. other formal statements appearing in items associated with the organisation
  3. other sources (including reference sources such as VIAF or Wikipedia).

When an organisation changes its name and the name change can be considered important, create a new record for the new name. When an organisation appears under a changed name, there have probably also been changes within the organisation or its operations. If an organisation has undergone extensive reorganisation, merged with others, dissolved into several, or if the business has changed direction significantly, the organisation should be considered new.

Register in each record, old and new, a Related organisation of the type earlier/later, an historical note explaining the relations and start and end time if the information exists.

It is possible, but not mandatory, to specify the authorised form in all three display languages ​​in cases where they are different, eg. Jesuits/Jesuittordenen/Jesuitorden.

The preferred name form must be in Latin script.

Do not enter more than one name for the same language.

Do not enter alternative name forms in this section.

Authority

Language code, display language:

Select the language corresponding to the name form, normally based on the location of your organisation.

Organisation name:

Enter the authoritative name form of the organisation corresponding to the selected language.

Example: Archaeological Museum of Thessaloniki (in English). (Do not enter Αρχαιολογικό Μουσείο Θεσσαλονίκης).

Subordinate unit:

For example used for:

  • an organisation with a name that includes words indicating that the organisation is part of another
  • an organisation with a name that includes words indicating administrative subordination
  • university faculty, department, laboratory, etc. with a name that only indicates a field of study

Example: Department of English (subordinate unit of University of Kansas).

Subordinate and related organisations that do not belong to any of the types above are authorised under their own names if the names are independent.

Example: Uppsala University Library (own authority record and not subordinate to Uppsala University).

Addition:

Geographical names as additions to names are used, for example, to distinguish organisations with the same name or for names of local departments, branches and the like that represent the activities of a parent organisation in a particular location.

Example: London (for Riviere & Sons).

Start date:

End date:

Enter the operating time for the organisation in start date and end date.

All types of time data are specified according to the ISO 8601 standard according to the pattern yyyy (year), yyyy-mm (year and month) and yyyy-mm-dd (year, month and day).

Example: 1707 (year) 05 (month) 13 (day).

Example: 0470 (year).

Era (BC):

Enter BC for years before Christ. Otherwise leave unselected.

Example: 0470 (year) + BC (the year 470 BC).

Year/date displayed

Enter uncertain and approximate time information as free text. Where applicable, the year of birth/death can also be entered above for better searchability.

Example: 1717? (? is used for probable year).

Example: 1717 or 1718 (or is used for alternative and immediately following years).

Example: ca 1717 (ca is used for approximate year. Here the time is more fluid and can cover a time span of up to ten years).

Example: ca 1717-1767 (for approximate timespan).

In all these cases, 1717 can also be entered as a numerical year of birth above, which means that 1717 will also be searchable as a year.

Example: Between 510 and 323 BC (as a time period).

Do not use this element if the exact date is known.

Variant

Language code:

Select the language corresponding to the name form. Any language can be specified. The 20 most common options are listed first. After these the others follow alphabetically.

Variant:

Select the type of variant form. Other is selected if none of the more specified ones are applicable.

Organisation name:

Enter the variant name form of the organisation corresponding to the selected language.

Example: Αρχαιολογικό Μουσείο Θεσσαλονίκης (translation in Greek).

Example: В.Ц.И.К (abbreviation in Russian for Всероссийский Центральный Исполнительный Комитет)

Subordinate unit:

Example: Swedish Department (subordinate unit of International exhibition 1862).

Addition:

Geographical names as additions to names are used, for example, to distinguish organisations with the same name or for names of local departments, branches and the like that represent the activities of a parent organisation in a particular location.

Example: Melbourne (for William Heinemann Ltd).

Text direction:

Select Right-to-left only if the name should be displayed from right to left, e.g, names in Arabic or Hebrew. Otherwise leave it unselected.

Note:

Type:

Select the type of note from the list. Enter only one note for each type (all information in one element).

  • General - General information for which a specialised note field has not been defined.
  • Biographical/historical - Biographical information about an individual or historical information about an institution.
  • Source - Citation for a consulted source in which information is found related in some manner to the entity represented by the authority record or related entities, for example a bibliographic reference or a URL.

Example: Swedish trading company that traded with East Asia, founded in Gothenburg in 1731 and formally dissolved in 1813. (historical).

Identifier:

Type:

Select the type of identifier from the list.

Example: 133471839 (VIAF for Ostindiska kompaniet).

Enter the number/code only, not the full URL. A link to the source will automatically be created on the web page.

Related organisation

Type of relation:

Select the type of relation to another organisation record in Alvin.

Search for the authority record for the related organisation and link to the organisation described in the record. Create a new organisation record if needed.

External electronic location (URL)

URL:

An external link that is relevant to the record described.

Example: https://www.uu.se

The URL must be correctly encoded according to the RFC 3986 standard and start with http:// or https://.

Example: https://www.uu.se

It is recommended that all URLs used with the element are persistent.

Display label:

A descriptive label displayed on the web page.

Example: Uppsala University
