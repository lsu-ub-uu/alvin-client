Place

Authority records for places are used in Alvin for:

  • Place of publication or manufacture of an object
  • Place of birth of a person
  • Place of death of a person
  • Place in a postal address of an organisation

Authority work means creating records as linked entities. This involves choosing a preferred name and, in some cases, distinctive additions to make the name unique. An important part of the work is to search for and check if an existing name form already exists as a linkable entity and, if so, link to it.

The aim is to be able to unambiguously identify these places regardless of what they were called during different time periods or in different languages.

A country (e.g. Sweden) should not be specified as an authority record for a place. However, land areas (e.g. Skåne, Öland, England), farms, churches, etc. can be specified if relevant to the above.

Although the main purpose of authority records for places is the above context, such a record can also be used as a subject heading in the same way as authority records for persons and organisations. Since places are not always as easily identifiable as the latter, it is often better to use the property Subject, Geographic coverage (free text) which is more flexible. For example, you can enter very specific places such as Drottninggatan 1, Uppsala or countries/areas regardless of status, e.g. Sweden, the Roman Empire, the Soviet Union, etc.). Likewise, all other types such as lakes, bridges, ancient remains, etc.

Sections in the registration form

At least one preferred name form of the place must be specified for the web pages in English, Swedish or Norwegian. This language specifies the language version of the web page rather than the language of the name form. If only one name form is specified, it will be displayed on all language versions of the web pages.

Find the current authoritative form in e.g. GeoNames or Wikipedia.

It is possible, but not mandatory, to specify the authorised form in all three display languages ​​in cases where they are different, e.g. Copenhagen/København/Köpenhamn.

Do not enter more than one name for the same language.

Do not enter alternative name forms in this section.

Authority

Language code, display language:

Select the language corresponding to the name form, normally based on the location of your organisation.

Place:

Enter the authoritative name form of the place corresponding to the selected language.

Example: Oslo. (Do not enter Christiania).

A name that is not considered the authoritative form, e.g. alternative spellings, previous variants or other languages. Specifying alternative names makes the resource searchable using these.

Variant

Language code:

Select the language corresponding to the name form. Any language can be specified. The 20 most common options are listed first. After these the others follow alphabetically.

Place:

Enter the alternative name form of the place corresponding to the selected language.

Multiple name forms can be entered for the same language.

Example: Christiania. (Alternative form for Oslo in Norwegian).

Example: Kristiania. (Another alternative form for Oslo in Norwegian).

Example: Asloa. (Oslo in Latin).

Text direction:

Select Right-to-left only if the name should be displayed from right to left, e.g, names in Arabic or Hebrew. Otherwise leave it unselected.

Country:

Select the country where the described place is currently located. The 20 most common options are listed first. After these the others follow alphabetically.

Countries and kingdoms that no longer exist as independent states cannot be specified here.

Latitude:

Enter the latitude of the place in WGS84 format (as compressed degrees with decimals, numbers only).

Example: 59.916667

Longitude:

Enter the longitude of the place in WGS84 format (as compressed degrees with decimals, numbers only).

Example: 10.75

Find the latitude and longitude in e.g. GeoNames or Wikipedia.

It is also possible to click on a point in the map in the registratation form to select the position. Selected values must be first deleted in order to select new ones from the map.
